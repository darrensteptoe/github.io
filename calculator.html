Campaign Probability & Risk Model

Mode Controls

Probability Simulation
Runs thousands of simulated elections to estimate win probability.
Margin Distribution
Shows average margin, worst case, best case, and volatility.
Sensitivity Analysis
Shows which variable (turnout, contact, persuasion, GOTV) moves probability most.
Diagnostics
Flags aggressive assumptions or structural deficits.
Institutional Anchoring Adjustment
Applies a structural filter if part of your persuasion universe isn’t truly movable.
Volatility Inputs
Controls how much uncertainty the simulation applies to each variable.
If you uncheck Probability Simulation, the tool becomes deterministic-only (no randomness). Distribution and sensitivity are disabled because they require simulation.

Core Inputs

Extra cushion above half + 1. Helps account for late breaks, noise, and execution variance.
Votes you believe you already have locked.

Persuasion & GOTV Inputs

This is the persuadable list BEFORE any “movable share” adjustment.
Use this if “undecided” voters are actually anchored to churches, clubs, unions, leaders, etc. Example: 70 means you believe only 70% of this universe is truly movable.

Volatility Inputs (Std Dev)

These control how much the model “wiggles” each variable across simulations. Higher = more uncertainty modeled. Lower = more stability assumed.
Tip: Start with DEFAULTS.md, sanity-check with EXAMPLES.md. If results feel “too stable,” increase volatility or enable Movable Share.
